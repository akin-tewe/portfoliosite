NEIGHBORHOOD DESIGN SYSTEM — NOTION AUDIT
==========================================
Product: Notion (notion.so)
Audited: March 12, 2026
Method: Live product analysis via authenticated session (dark mode)
Views analyzed: 9 distinct screens
Auditor: Claude (conversation with Akin Tewe)

Purpose: This document captures every observed UI pattern from the live
Notion product, organized by abstraction layer. It serves as a working
reference for the Neighborhood design system component extraction. Stripe
and Discord audits will follow in separate documents. A cross-product
synthesis will be produced after all three audits are complete.


TABLE OF CONTENTS
=================
1. Page-level patterns (full-page structural templates)
2. Compositional patterns (mid-level grouped components)
3. Atomic components (individual UI elements with observed states)
4. Interaction patterns (behaviors and state transitions)
5. Notable design decisions
6. Extraction candidates for Neighborhood


========================================================================
1. PAGE-LEVEL PATTERNS
========================================================================
These are the full-page structural templates that define how Notion
organizes information at the highest level. Each represents a reusable
layout architecture that could be abstracted into a Neighborhood
layout component.

------------------------------------------------------------------------
1A. APP SHELL
------------------------------------------------------------------------
Source: Every screen observed
Observed in: All 9 views

Structure (left to right):
  [Sidebar] [Main Content Area] [Optional Right Panel]

Sidebar:
  - Fixed width (~240px estimated), dark background (slightly lighter
    than the main content bg in dark mode)
  - Collapsible: clicking the hamburger icon at top-left fully hides
    the sidebar, and the main content area expands to fill the space.
    A small hamburger icon remains at the top-left of the content area
    to restore it.
  - Organized into distinct vertical sections, separated by labeled
    group headers in muted uppercase text:
      * Top zone: Workspace selector (avatar + workspace name + chevron
        + edit/checkmark icon), then primary nav items (Search, Home,
        Meetings, Notion AI, Inbox, Library) — each is icon + label.
      * "Recents" section: Recently visited pages with page emoji/icon.
      * "Agents" section (labeled "Beta"): Agent items + "New agent"
        action + "More" expandable.
      * "Private" section: User's private pages, each with page
        emoji/icon. Includes "+ Add new" action at bottom.
      * "Teamspaces" section: Teamspace items with house icon. Includes
        "+ Add new" action.
      * "Notion apps" section: Notion Mail, Notion Calendar, Notion
        Desktop — each with distinct product icon.
      * Bottom zone (pinned, always visible): Settings, Marketplace,
        Trash — each with icon.
  - Active page indicated by: highlighted background fill on the
    sidebar item.
  - Hover behavior: items show slightly lighter background. On page
    items, hover reveals action icons (three-dot menu, plus icon) to
    the right of the item name.
  - Nesting: Teamspace items can expand to show child pages (observed
    "Akin Tewe's Space HQ" with expand arrow).

Top bar:
  - Contextual to the current page — NOT a global persistent bar.
  - Spans the width of the main content area.
  - Left side: Breadcrumb showing page emoji/icon + page name +
    privacy indicator (lock icon + "Private" with dropdown chevron).
    In split view, navigation arrows (< >) appear.
  - Right side: "Edited Xm ago" timestamp in muted text, "Share"
    button (icon + text + dropdown chevron), star/favorite icon,
    three-dot actions menu icon. Farther right: "New AI chat"
    dropdown, and panel toggle icons.
  - In Home view: simplified — only three-dot menu on far right.

Main content area:
  - Fluid width, centered content with max-width constraint.
  - Scroll behavior: the top bar stays fixed, content scrolls below.
  - Background: consistent dark surface across all views observed.

Right panel (AI chat):
  - Docked to the right edge of the viewport.
  - Contains: avatar icon, greeting message, suggested prompt cards.
  - Has its own top bar with "New AI chat" dropdown.
  - Does NOT displace the main content — overlays on top of it.
  - Can be toggled on/off via icons in the top-right corner.

Extraction potential for Neighborhood:
  → AppShell component: sidebar slot + topbar slot + content slot +
    optional right panel slot.
  → Sidebar component: section groups with labels, nav items with
    icons, active state, hover reveal actions, collapsible nesting.
  → TopBar component: breadcrumb + right-aligned actions zone.

------------------------------------------------------------------------
1B. HOME PAGE LAYOUT
------------------------------------------------------------------------
Source: notion-home (Home view)

Structure (top to bottom):
  [Greeting header]
  [Section: Recently visited — card row]
  [Section: Learn — card row with thumbnails]
  [Section: Upcoming events — split layout]
  [Section: Home views — database preview]

Greeting header:
  - Large, bold, centered text: "Good evening"
  - No additional metadata or subtitle

Section pattern (repeats 4 times):
  - Section header: icon (muted) + label text (muted, smaller size)
  - Content area: varies per section type (cards, events, database)
  - No visible dividers between sections — spacing alone creates
    separation

"Recently visited" section:
  - Horizontal row of compact cards (4 visible + 1 "New page" card)
  - Card anatomy: page icon/emoji at top-left, page title below,
    metadata line (triangle icon + "22m ago") at bottom
  - Cards have subtle border, rounded corners, same bg as page
  - "New page" card: plus icon centered, "New page" label, different
    visual treatment (muted/dashed appearance)
  - Cards appear to be horizontally scrollable if more exist

"Learn" section:
  - Horizontal row of media cards (4 visible, possibly scrollable)
  - Card anatomy: illustration/thumbnail image filling top ~60% of
    card, title text below image, metadata line at bottom (book icon +
    "2m read" or clock icon + "2m watch")
  - Cards are larger than the "Recently visited" cards
  - Illustrations are custom, editorial-style drawings (not photos)

"Upcoming events" section:
  - Split layout: left side shows a promotional card (calendar
    integration prompt with icon, description, "Connect Notion
    Calendar" link), right side shows event list items
  - Event list item anatomy: date label on left ("Today Mar 12" or
    "Fr Mar 13"), event title + time + location on right ("Team
    standup / 9 AM · Office"), optional action button ("Join and
    take notes" with video icon)
  - Items stacked vertically, separated by subtle horizontal lines

"Home views" section:
  - Inline database preview widget
  - Left side: icon + description text + "Select database" link
  - Right side: mini table with columns (Activity, Status) showing
    task rows with status text ("Done", "In progress", "Not started")
  - This appears to be an embedded database view, not a separate
    component

Extraction potential for Neighborhood:
  → SectionHeader component: icon + label, used as a repeating
    content organizer.
  → Card variants: CompactCard (icon + title + metadata) and
    MediaCard (thumbnail + title + metadata) — two distinct card
    patterns in the same view.
  → EventListItem: date column + content column with title, time,
    and action.
  → PromotionalCard / CTACard: icon + title + description + action
    link, used for feature promotion.

------------------------------------------------------------------------
1C. DATABASE LIST PAGE
------------------------------------------------------------------------
Source: notion-database-list (To Do List view)

Structure (top to bottom):
  [Page emoji/icon — large, decorative]
  [Page title — large heading]
  [View tabs + Toolbar row]
  [List items]
  [+ New task action]

Page emoji/icon:
  - Large emoji displayed above the page title as a decorative header
    element. In this case: a green checklist emoji rendered at ~48px+.
  - Positioned at the left margin of the content area.

Page title:
  - Large heading text ("To Do List"), bold, full width.
  - Appears to be editable inline (cursor changes on hover).

View tabs + Toolbar (same row):
  - Left side: View tabs — "To Do" and "Done", each prefixed with a
    small view-type icon (list icon for "To Do", checkbox icon for
    "Done"). Active tab has a subtle underline or bold treatment.
  - Right side: Toolbar icons — filter funnel, sort arrows, lightning
    bolt, search magnifier, properties slider, layout grid. These are
    icon-only buttons, muted by default.
  - Far right: "New" button — blue background, white text, with a
    dropdown chevron indicating it's a split button (clicking the
    chevron likely opens template options).

List items:
  - Each item is a full-width row.
  - Left side: Checkbox (square, rounded corners, blue fill when
    checked with white checkmark).
  - Center: Item text/title, regular weight.
  - Right side: Date value ("Today" or "Tomorrow"), muted text,
    right-aligned.
  - Rows are separated by subtle horizontal lines.
  - Selected/active row: highlighted with a blue left border accent
    (observed in split view mode).
  - Hover behavior: likely reveals action icons (not captured in
    static screenshot but consistent with Notion patterns).
  - Checked items: the checkbox shows a filled blue state with a
    white checkmark, and the text may show strikethrough (observed
    on "Check the box to mark items as done" row).

"+ New task" action:
  - Positioned below the last list item.
  - Muted text with a plus icon prefix.
  - Serves as an inline creation action — not a button, more of an
    action link.

Extraction potential for Neighborhood:
  → PageHeader component: emoji/icon + title + optional subtitle.
    Distinct from the top bar — this is the in-content page header.
  → ViewTabs: tabs with icon prefixes, used to switch between views
    of the same data. Distinct from navigation tabs.
  → Toolbar: row of icon-only action buttons + a primary action
    button. Right-aligned, horizontally scrollable potentially.
  → ListItem: checkbox + text + right-aligned metadata. A reusable
    row pattern.
  → InlineAction: muted text link with plus icon for inline creation.

------------------------------------------------------------------------
1D. DETAIL / SPLIT VIEW
------------------------------------------------------------------------
Source: notion-detail-split (clicked into a to-do item)

Structure:
  [Left panel: narrowed database list]
  [Right panel: item detail page]

Left panel (narrowed list):
  - Same as the database list page but narrower width.
  - Items remain visible and interactive.
  - Selected item indicated by blue left border accent.
  - Panel has its own toolbar row (view tabs + icons + New button).
  - Navigation arrows appear at the top of the detail panel for
    moving between items.

Right panel (detail page):
  Structure (top to bottom):
    [Item title — large, editable heading]
    [Property rows]
    [Collapsible "more properties" toggle]
    [Comments section]
    [Page body content]
    [Callout block]

  Property rows:
    - Each row: icon + property label (muted) + property value
    - "Due date" row: calendar icon + "Due date" label + "Today"
      value text
    - "Status" row: sparkle/status icon + "Status" label + "Not
      started" pill badge (gray background, rounded, small text)
    - Property label and value are arranged in a two-column layout
      with consistent spacing.
    - Values can be different types: plain text, date, status badge,
      person, URL, etc.

  Collapsible properties:
    - "v 1 more property" — a toggle link that expands to reveal
      additional property rows. Chevron rotates on expand.

  Comments section:
    - Header: "Comments" label
    - Input: Avatar circle (initial "A") + "Add a comment..."
      placeholder input
    - This is an inline comment composer, not a modal.

  Page body:
    - Rich text content area with block-based editing.
    - Contains: paragraphs, checkboxes, inline page links
      (icon + "Example sub-page!" as a clickable link within text).
    - Content is editable inline.

  Callout block:
    - Tinted background (blue-ish tint, distinct from page bg).
    - Left edge: icon (Notion logo "N" icon).
    - Content: text with bold segments + nested clickable link.
    - This is a "callout" or "aside" block — a contained informational
      element within the page body.
    - Specific example observed: informational callout about Tasks
      database configuration with a nested action link.

  Top bar in split view:
    - Additional navigation controls: expand arrows, split view
      toggle icon, up/down arrows for navigating between items.
    - "Add icon", "Add cover", "Customize layout" action links appear
      above the title in a muted toolbar.

Extraction potential for Neighborhood:
  → SplitView layout: list panel + detail panel, with the list
    narrowing and the selected item highlighted.
  → PropertyRow: icon + label + typed value. Highly reusable — any
    key-value display with an icon prefix.
  → PropertyGroup: collapsible group of PropertyRows with a toggle.
  → CommentInput: avatar + inline text input, used for comment
    composition.
  → CalloutBlock: tinted container with icon + rich text content.
    Could become an Alert or Callout component.
  → InlinePageLink: icon + text link for referencing other pages
    within content.

1E. SETTINGS MODAL
------------------------------------------------------------------------
Source: notion-settings-preferences, notion-settings-people

Structure:
  [Modal overlay with backdrop dim]
  [Modal container]
    [Left sidebar — settings navigation]
    [Right content — settings page]
  [Close button — X in top-right corner]

Modal overlay:
  - Semi-transparent backdrop dimming the page behind.
  - Modal is large — takes up most of the viewport (~80% width,
    ~90% height estimated).
  - Centered both horizontally and vertically.
  - Close: X button in top-right corner of modal, or click backdrop,
    or press Escape.

Settings sidebar (left):
  - Grouped navigation items under labeled sections:
    * "Account" section: avatar + name ("Akin Tewe"), Preferences,
      Notifications, Connections
    * "Workspace" section: General, People, Import
    * "Features" section: Notion AI, Public pages, Emoji
    * "Integrations" section: Connections
    * "Admin" section: Teamspaces, Security, Identity
    * "Access & billing" section: "Upgrade plan" with blue circle
      accent icon
  - Each nav item: icon + label text.
  - Active item: highlighted background, bolder text.
  - Section labels: muted, uppercase or small-caps, act as
    non-interactive group headers.

Settings page — Preferences (right panel):
  Structure:
    [Breadcrumb: "Account"]
    [Page title: "Preferences"]
    [Page description: "Choose how you want Notion to look and behave"]
    [Setting sections, stacked vertically]

  Settings page header:
    - Breadcrumb (muted text, indicating parent category).
    - Title (large, bold).
    - Description (muted, regular weight, below title).
    - This header pattern is consistent across all settings pages.

  Setting sections — each section has:
    - Section header: bold text label ("Appearance", "Language & Time",
      "Desktop app", "Privacy").
    - One or more setting rows below the header.
    - Sections separated by generous vertical spacing (no visible
      divider lines — spacing alone).

  Setting row anatomy (THE KEY PATTERN):
    - Left side: Bold label text + muted description text below it.
    - Right side: Control element, vertically centered against the
      label+description block.
    - Control types observed:
      * Select/dropdown: "Use system setting v", "English (US) v",
        "Relative v", "(GMT-4:00) New York v",
        "Last visited page v", "Customize v"
      * Toggle/switch: binary on/off (observed both states — off is
        muted track, on is blue track with white thumb shifted right).
    - Full-width row with consistent horizontal padding.
    - Rows within a section are separated by subtle horizontal lines.

  Setting rows observed on Preferences page:
    Appearance section:
      - "Theme" / "Choose a theme for Notion on this device" → Select
    Language & Time section:
      - "Language" / "Choose the language you want to use Notion in" → Select
      - "Always show text direction controls" / [long description] → Toggle (off)
      - "Start week on Monday" / [description] → Toggle (off)
      - "Date format" / "Set the default format for new @date mentions" → Select
      - "Set time zone automatically using your location" / [description] → Toggle (on, blue)
      - "Time zone" / "Choose your time zone" → Select
    Desktop app section:
      - "Open links in desktop app" / [description] → Toggle (off)
      - "Open on start" / [description] → Select
    Privacy section:
      - "Cookie settings" / "See the Cookie Notice for more information" → Button

Settings page — People (right panel):
  Structure:
    [Breadcrumb: "Workspace"]
    [Page title: "People"]
    [Page description + "Learn more" link]
    [External link: "View People Directory" with arrow icon]
    [Add members row]
    [Tabs + search + action button]
    [Content area / empty state]

  Page header:
    - Same pattern as Preferences: breadcrumb + title + description.
    - Additional element: "Learn more" link (blue, inline with
      description text).
    - External link: "View People Directory" with arrow icon —
      right-aligned, with external link arrow icon.

  Add members row:
    - Label: "Add members via link" + description text.
    - Controls: "Copy link" button (outlined, with copy icon) +
      Toggle switch — side by side on the right.

  Tabs + action bar:
    - Horizontal tabs: "Guests" (active, filled bg), "Members 1"
      (with count), "Groups", "Contacts".
    - Right side: Search icon, "Add members" button with dropdown
      chevron (blue primary).
    - Tab style: pill/chip style with filled background on active tab,
      not underline style. Different from the view tabs on database
      pages.

  Empty state:
    - Dashed border container taking up the remaining content area.
    - Centered content: person icon, "No guests yet" text,
      "Import contacts" action link.
    - Clean, intentional empty state — not just blank space.

Extraction potential for Neighborhood:
  → SettingsModal: large modal with sidebar nav + content area.
  → SettingsNav: grouped navigation with section labels and items.
  → SettingsPageHeader: breadcrumb + title + description.
  → SettingsSection: section label + group of setting rows.
  → SettingRow: THE most reusable mid-level component from this
    audit. Label + description + right-aligned control. The control
    slot accepts Toggle, Select, Button, or custom content.
  → EmptyState: dashed container + centered icon + text + action.
  → TabBar (pill variant): distinct from underline-style tabs.

1F. COMMAND PALETTE / SEARCH OVERLAY
------------------------------------------------------------------------
Source: notion-command-palette (Ctrl+K)

Structure:
  [Overlay with subtle backdrop]
  [Search input bar]
  [Filter chip row]
  [Results panel (left) + Preview panel (right)]
  [Footer with keyboard hints]

Overlay:
  - Centered, large panel (~70% viewport width, ~60% height).
  - Subtle backdrop dim, less aggressive than settings modal.
  - Dismisses on Escape or backdrop click.

Search input:
  - Full-width input at top of panel.
  - Left: magnifying glass icon.
  - Placeholder: "Search or ask a question in Akin Tewe's Space..."
  - Right: toggle icons (likely search mode switches).
  - No visible border — the input is flush with the panel top.

Filter chip row:
  - Below the search input, horizontal row of filter chips.
  - Chips observed:
    * "Aa Title only" — icon + label, acts as a toggle filter.
    * "Created by" with chevron — icon + label + chevron, opens a dropdown.
    * "In" with chevron — icon + label + chevron, opens a scope dropdown.
    * "+ Filter" — add action to add more filter chips.
  - Chips have subtle border, rounded pill shape, muted bg.
  - Mixed types: toggle chips, dropdown chips, action chip.

Results panel (left):
  - Section label: "Today" — groups results by time.
  - Result items: page emoji/icon + page name, each on its own row.
  - Selected/hovered item: highlighted background.
  - Timestamp: triangle icon + relative time visible on some items.

Preview panel (right):
  - Shows a preview of the selected result.
  - Contains: page emoji (large), page title, and body content
    preview (checklist items, text blocks).
  - Has its own action icons in top-right (expand, external link).
  - Bordered or visually separated from the results panel.
  - Scrollable independently of the results list.

Footer:
  - Left: "Ctrl+Enter Open in new tab" keyboard shortcut hint.
  - Right: Settings gear icon.
  - Muted text, small size.

Extraction potential for Neighborhood:
  → CommandPalette: a composed overlay component with specific
    structural requirements (search + filters + results + preview
    + footer).
  → FilterChipRow: horizontal row of mixed chip types.
  → SearchInput: icon + input + action icons, no visible border.
  → ResultsList: grouped, selectable list with section headers.
  → PreviewPane: content preview alongside a selection list.

2. COMPOSITIONAL PATTERNS
========================================================================
These are mid-level component compositions — groups of elements that
appear together consistently and could be extracted as single reusable
components.

------------------------------------------------------------------------
2A. SETTING ROW
------------------------------------------------------------------------
Sources: Settings > Preferences, Settings > People

The most frequently repeated compositional pattern in the audit.
Appears 10+ times on the Preferences page alone.

Anatomy:
  +-------------------------------------------------------------+
  | [Bold label text]                           [Control]        |
  | [Muted description text, possibly wrapping                   |
  |  to multiple lines]                                          |
  +-------------------------------------------------------------+

  - Label: bold/medium weight, primary text color.
  - Description: regular weight, muted/secondary text color. Can wrap
    to 2+ lines. Positioned directly below the label with minimal gap.
  - Control: right-aligned, vertically centered against the full
    label+description block. Types observed:
    * Toggle/Switch (boolean)
    * Select/Dropdown (choice from options)
    * Button (action trigger)
  - Separator: subtle horizontal line below each row.
  - Full-width within the settings content area.

Variations observed:
  - Toggle variant: label + description → Toggle switch
  - Select variant: label + description → Select dropdown
  - Button variant: label + description → Button
  - Compound variant: label + description → Button + Toggle side by
    side (two controls in one row, observed in "Add members via link")

------------------------------------------------------------------------
2B. MEMBER ROW
------------------------------------------------------------------------
Sources: Share popover, Settings > People

Anatomy:
  +-------------------------------------------------------------+
  | [Avatar] [Name]                          [Role dropdown]     |
  |          [Email/metadata, muted]                             |
  +-------------------------------------------------------------+

  - Avatar: circular, shows initial letter on colored background.
    Estimated 32px diameter.
  - Name: bold/medium weight, primary text color.
  - Metadata: email address or additional info, muted text, below name.
  - Role control: right-aligned, dropdown showing current role
    ("Full access" with chevron).
  - This pattern appears in both the Share popover and the People
    settings page.

------------------------------------------------------------------------
2E. ACTIONS DROPDOWN (RICH MENU)
------------------------------------------------------------------------
Source: Three-dot menu on any page

The most complex single component observed in the audit. Contains at
least 6 distinct item types within a single dropdown.

Anatomy (top to bottom):
  +-------------------------------------------+
  | [Search actions...]       <- searchable    |
  |-------------------------------------------|
  | [Ag Default] [Ag Serif] [Ag Mono]         |
  |  <- segmented font selector               |
  |-------------------------------------------|
  | [Icon] Copy link              Ctrl+Alt+L  |
  | [Icon] Copy page contents                 |
  | [Icon] Duplicate              Ctrl+D      |
  | [Icon] Move to                Ctrl+Shift+P|
  | [Icon] Move to Trash                      |
  |-------------------------------------------|
  | [Icon] Small text                 [Toggle]|
  | [Icon] Full width                 [Toggle]|
  | [Icon] Table of contents          [Toggle]|
  | [Icon] Customize layout                   |
  | [Icon] Lock page                  [Toggle]|
  |-------------------------------------------|
  | [Icon] Use with AI                     >  |
  | [Icon] Suggest edits                   >  |
  | [Icon] Translate                       >  |
  |-------------------------------------------|
  | [Icon] Undo                       Ctrl+Z  |
  | [Icon] Import                              |
  | [Icon] Export                              |
  |-------------------------------------------|
  | [Icon] Updates & analytics                 |
  | [Icon] Version history                     |
  | [Icon] Notify me             Important     |
  | [Icon] Connections           None       >  |
  | [Icon] Open in Windows app                 |
  |-------------------------------------------|
  | Word count: 104 words                      |
  | Last edited by Akin Tewe                   |
  | Today at 8:13 PM                           |
  +-------------------------------------------+

Item types within this single dropdown:
  1. Search input — filters the menu items.
  2. Segmented selector — 3 font options shown as a horizontal row
     of styled text samples. Active one indicated.
  3. Action items — icon + label + optional keyboard shortcut
     (right-aligned, muted).
  4. Toggle items — icon + label + toggle switch (right-aligned).
     Stateful — reflects current page setting.
  5. Submenu items — icon + label + chevron indicating a submenu.
  6. Metadata items — icon + label + right-aligned value text.
  7. Footer metadata — non-interactive, muted text showing page
     stats (word count, last editor, timestamp).

Dividers: horizontal lines separate each group of related items.

  - Member list: reuses the MemberRow pattern.
  - Access section: icon + description + dropdown control.
  - Footer: help link + action button.

========================================================================
4. INTERACTION PATTERNS
========================================================================

4A. SIDEBAR HOVER REVEAL
  When hovering over a page item in the sidebar, action icons (three-dot
  menu, plus/add icon) appear to the right of the item name. Hidden by
  default, visible only on hover.

4B. SPLIT VIEW SELECTION
  Clicking a list item opens a split view: list narrows left, detail
  opens right. Selected item indicated by blue left border accent.
  Up/down arrows allow navigating between items.

4C. COMMAND PALETTE PREVIEW
  As the user navigates results (arrow keys or hover), the preview
  pane updates to show content of the selected result. Live preview.

4D. MODAL VS POPOVER DISTINCTION
  Settings = modal (centered, large, backdrop). Share = popover
  (anchored to trigger, small, subtle/no backdrop). Two distinct
  overlay patterns.

4E. INLINE EDITING
  Page titles and content are editable inline — no modal or separate
  editing view. Properties in detail view also inline-editable.

4F. COLLAPSIBLE SECTIONS
  Multiple levels: sidebar groups, property rows ("1 more property"),
  toggle blocks in content, category groups in settings sidebar.


========================================================================
5. NOTABLE DESIGN DECISIONS
========================================================================

5A. DARK MODE AS DEFAULT EXPERIENCE
  All surfaces use dark grays, not true black. Text hierarchy achieved
  through opacity/brightness of white text.

5B. MINIMAL BORDERS
  Separation primarily through spacing and subtle background
  differences. When borders appear, they are extremely subtle — 1px
  at very low opacity.

5C. ICON CONSISTENCY
  Every interactive element has an associated icon. Outlined style
  with consistent stroke weight throughout.

5D. CONTROL ALIGNMENT
  In settings, controls always right-aligned with label+description
  on left. Consistent scan pattern.

5E. MIXED TAB STYLES
  At least two visually distinct tab styles (underline and pill).
  Suggests tabs are a component family with style variants.

5F. NO TRADITIONAL FORM LAYOUTS
  Very few label-above-input patterns. Most configuration done through
  toggles and selects in setting rows. Text inputs mainly in search
  and invite contexts.

5G. POPOVER VS MODAL DISTINCTION
  Clear separation: modals for complex workflows (Settings), popovers
  for quick actions (Share). Two separate patterns.

5H. SECTION ORGANIZATION WITHOUT HEAVY CHROME
  Sections organized by spacing and muted headers, not by cards or
  heavy borders. The content itself creates the visual structure.

6. EXTRACTION CANDIDATES FOR NEIGHBORHOOD
========================================================================
Ranked by reusability and demonstration value for a design system.

TIER 1 — HIGH PRIORITY (compositional, demonstrate systems thinking):
  - SettingRow: label + description + control slot. Massively reusable.
  - ActionMenu / RichDropdown: searchable, multi-type content dropdown.
  - CommandPalette: search + filters + results + preview overlay.
  - EmptyState: dashed border + centered content pattern.
  - PropertyRow: icon + label + typed value row.
  - CalloutBlock: icon + tinted container + rich content.

TIER 2 — STRONG CANDIDATES (compositional, good demo value):
  - MemberRow: avatar + name/metadata + role control.
  - SectionHeader: icon + label content divider.
  - FilterChipRow: mixed chip types in a horizontal row.
  - PageHeader: emoji + title (in-content, not nav).
  - TabBar: with underline and pill variants.
  - EventListItem: date column + content column.

TIER 3 — ATOMIC FOUNDATION (necessary but less distinctive):
  - Button (primary, secondary, ghost, icon-only, split).
  - Toggle/Switch.
  - Select/Dropdown trigger.
  - Checkbox (checked/unchecked).
  - Badge/Pill (status variant with dot prefix).
  - Avatar (initials variant).
  - Input (search variant, inline variant).
  - Divider/Separator.
  - Icon system.

cross-product patterns emerge.
