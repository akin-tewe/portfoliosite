NEIGHBORHOOD DESIGN SYSTEM — DISCORD AUDIT
===========================================
Product: Discord (discord.com/app)
Audited: March 13, 2026
Method: Live product analysis via authenticated session (dark mode)
Server used: RDCWorld (large, active community server with roles,
  categories, voice channels, bots, and custom emoji)
Views analyzed: 12 distinct screens
Auditor: Claude (conversation with Akin Tewe)

Purpose: This document captures every observed UI pattern from the live
Discord product, organized by abstraction layer. It serves as a working
reference for the Neighborhood design system component extraction. This
is the second of three product audits (Notion complete, Stripe pending).
A cross-product synthesis will be produced after all three audits.


TABLE OF CONTENTS
=================
1. Page-level patterns (full-page structural templates)
2. Compositional patterns (mid-level grouped components)
3. Atomic components (individual UI elements with observed states)
4. Interaction patterns (behaviors and state transitions)
5. Notable design decisions
6. Extraction candidates for Neighborhood


========================================================================
1. PAGE-LEVEL PATTERNS
========================================================================

------------------------------------------------------------------------
1A. APP SHELL (THREE-COLUMN)
------------------------------------------------------------------------
Source: Every screen observed

Discord's app shell is the most complex of the three audited products.
It uses a persistent three-column layout, unlike Notion's two-column.

Structure (left to right):
  [Server Strip] [Channel Sidebar] [Main Content Area]

Server strip (far left, ~72px wide):
  - Vertical column of circular server icons/avatars.
  - Each server is a circular image (~48px) with rounded square on
    hover transition.
  - Active server: white left border pill indicator (vertical bar on
    the left edge of the icon, ~40px tall).
  - Hovered server: smaller white left border pill (~20px tall).
  - Notification badges: red circles overlapping bottom-right of
    server icon, showing unread count number in white text.
  - Some servers have animated icons (GIF avatars).
  - Dividers: thin horizontal lines separate logical server groups
    (e.g., DMs at top, then servers, then "Add Server" and "Discover"
    icons at bottom).
  - Bottom zone: "Add a Server" (green + icon) and "Explore
    Discoverable Servers" (compass icon) buttons.
  - Scrollable: when many servers, the strip scrolls vertically.
  - This strip is ALWAYS visible — it never collapses.

Channel sidebar (~240px wide):
  - Top zone: Server name + dropdown chevron (clickable, opens server
    dropdown menu). Below that: server banner image (if set), "Boost
    Goal" progress bar with boost count, utility links ("4 Events",
    "Channels & Roles").
  - Channel list: organized under collapsible category headers.
    * Category header anatomy: emoji + category name in uppercase +
      collapse chevron + optional action icon (e.g., person+ icon).
      Clicking collapses/expands the category's channels.
    * Text channel: # icon + channel name + optional emoji suffix.
    * Voice channel: speaker icon + channel name + optional emoji.
      Shows connected user count (e.g., "00 | 15" meaning 0 speaking,
      15 connected) and a settings gear icon.
    * Announcement channel: megaphone icon + channel name. Some have
      alert (!) indicator.
    * Forum channel: forum icon + channel name.
    * Stage channel: stage icon + "EVENT STAGE" label.
    * Rules channel: shield/checkbox icon + channel name.
  - Active channel: highlighted background (lighter than sidebar bg),
    bold text.
  - Unread channels: bold white text (vs. muted gray for read).
  - Notification badges: red circles with count, right-aligned per
    channel.
  - "Suggested" section: appears at top of channel list with a dismiss
    (X) icon, showing recommended channels with unread badges.
  - "Show All" button: at very bottom of sidebar, for servers with
    hidden/muted channels.
  - Bottom user panel: persistent bar below the channel list (see
    compositional pattern 2H).

Main content area:
  - Top: Channel header bar (see compositional pattern 2G).
  - Center: Content area — changes based on channel type. For text
    channels: message feed. For voice: voice UI. For forums: thread
    list.
  - Bottom: Message input bar (see compositional pattern 2F).
  - Optional right panel: Member list (toggleable via icon in channel
    header). Shows server members grouped by role with online status.

Extraction potential for Neighborhood:
  → Three-column AppShell: server strip slot + sidebar slot + content
    slot. More complex than Notion's two-column.
  → ServerStrip: vertical icon navigation with active indicators,
    notification badges, and grouped sections. Unique pattern not seen
    in Notion.
  → ChannelSidebar: collapsible categories, multiple channel types
    with distinct icons, notification states, user panel at bottom.
  → The three-column layout itself is a strong extraction — it
    demonstrates that Neighborhood's layout components can handle
    varying complexity levels.

1B. TEXT CHAT VIEW
------------------------------------------------------------------------
Source: #rdc-gang channel, #variety-fighters channel

Structure (top to bottom):
  [Channel header bar]
  [Banner: "X+ new messages since..." — conditional]
  [Message feed — scrollable]
  [Typing indicator — conditional]
  [Message input bar]

Banner (conditional):
  - Yellow/gold background, spans full width.
  - Text: "13+ new messages since 1:20 PM on March 11, 2026"
  - Clickable — scrolls to the unread boundary.
  - Right side: "Mark As Read" action with dismiss icon.
  - Also observed: "This channel is not on your channel list" info
    banner with lighter background + "+ Add to Channel List" button.

Message feed:
  - Vertically scrolling list of messages.
  - Messages grouped by user: first message in a group shows full
    anatomy (avatar + username + timestamp + body), subsequent
    messages from the same user within a short time window show only
    the body (no avatar/username repeated), with a hover-revealed
    timestamp.
  - Infinite scroll — loads older messages on scroll up.
  - New message boundary indicated by a red line with "NEW" label.

Typing indicator:
  - Appears below the last message, above the input bar.
  - Animated dots (three bouncing dots) + "[username] is typing..."
  - Multiple users: "User1 and User2 are typing..."

Extraction potential for Neighborhood:
  → ChatView layout: header + conditional banners + scrollable feed
    + typing indicator + input bar.
  → Banner/AlertBar: dismissible, full-width, colored background,
    with text + action. Multiple variants (warning/gold for unread,
    info/muted for channel suggestions).
  → TypingIndicator: animated dots + text. Small but distinctive.

1C. USER SETTINGS (FULL-PAGE MODAL)
------------------------------------------------------------------------
Source: Settings > My Account, Notifications, Appearance

Structure:
  [Full-page overlay replacing the app shell]
  [Left sidebar — settings navigation]
  [Right content — settings page]
  [Close button — X icon with ESC label, top-right]

Key difference from Notion: Discord's settings is a FULL-PAGE takeover,
not a modal overlaying the app. The entire app shell is hidden. Only the
close button (X with "ESC" text) returns you to the app.

Settings sidebar:
  - User profile card at top: avatar + display name + "Edit Profiles"
    link with pen icon.
  - Search input: magnifying glass + "Search" placeholder.
  - Grouped navigation under section labels:
    * "User Settings": My Account, Content & Social, Data & Privacy,
      Family Center, Authorized Apps, Devices, Connections,
      Notifications
    * "Billing Settings": Nitro, Server Boost, Subscriptions, Gift
      Inventory, Billing
    * "App Settings": Appearance, Accessibility, Voice & Video, Chat
      (with pink "NEW" badge), Keybinds, Language & Time, Streamer
      Mode, Advanced
    * "Activity Settings": Activity Privacy
    * Log Out (red text, at bottom)
  - Active item: highlighted background, white text (vs. muted for
    inactive).
  - Some items expand sub-navigation: Notifications expands to show
    Overview / Sounds / Badges / Email / Advanced as indented child
    items below the parent.
  - Section labels: muted uppercase text, non-interactive.
  - Footer: version info ("stable 307958") + links (Privacy Policy,
    Terms of Service, More).

Settings content area patterns:

  My Account page:
    - Tabs at top: "Security" / "Standing" (underline style, active
      has blue underline + bold text).
    - Profile card: banner image + large avatar overlapping banner +
      display name + "Edit User Profile" button (outlined).
    - Below avatar: row of activity/status badge icons (emoji-sized,
      various colors).
    - Property rows (DISTINCT from Notion's):
      * "Display Name" label (muted, above) + "paid actor" value
        (primary text, below) + "Edit" button (right-aligned).
      * "Username" / "eightybot" / Edit
      * "Email" / masked "********@gmail.com" + "Reveal" link (blue)
        / Edit
      * "Phone Number" / masked "********6675" + "Reveal" / "Remove"
        + "Edit" (two action buttons)
    - Rows separated by subtle dividers.
    - "Password and Authentication" section:
      * Section header (large, bold)
      * "Change Password" button (blue/primary, filled)
      * "Authenticator App" subsection: bold label + description
        paragraph + "Enable Authenticator App" button (blue/primary)
      * "Security Keys" subsection following same pattern.

  Notifications page:
    - Sub-navigation in sidebar: Overview, Sounds, Badges, Email,
      Advanced — indented under Notifications parent item.
    - "Overview" section header (large, bold).
    - Description paragraph below header (muted).
    - "Notify me when..." subheader (medium weight).
    - Toggle rows:
      * Label text (primary) + toggle switch (right-aligned).
      * Some with description text below label.
      * Toggle on: blue track. Toggle off: gray track.
    - Select row: "Someone reacts to my messages" label + "All
      Messages" dropdown with chevron.
    - "Sounds" section:
      * Toggle rows with additional "Preview Sound" link (blue,
        below the label+description).
    - "Related Settings" section:
      * Cross-link row: icon + "Voice & Video" title + description +
        chevron (right-aligned). Links to another settings page.

  Appearance page:
    - Live preview: miniature chat view at top showing how the current
      theme looks with sample messages. This is a real rendered preview,
      not a static image.
    - Sub-navigation: Theme / In-app Icon / UI Density / Message
      Spacing / Scaling (indented in sidebar).
    - "Theme" section:
      * "Default Themes" label + description + "Preview Themes"
        button (blue/outlined).
      * Theme swatch row: 4 square swatches (representing dark themes
        of varying darkness) + a sync/system icon. Selected swatch
        has a visible border or highlight.
    - Promotional card: illustration + "Make your Discord look
      *juuusst* right" title + description + "Try it out" button.
      This is a Nitro upsell embedded in settings.
    - "Color Themes" section:
      * Grid of gradient color swatches (~20 options in 3 rows).
      * Selected swatch: white checkmark overlay.
      * "NEW" badge (pink/red pill) on the first swatch.
      * Some swatches locked (Nitro-only), indicated by lock icon or
        different treatment.

Extraction potential for Neighborhood:
  → Full-page settings layout (alternative to Notion's modal approach).
  → Settings sidebar with search, profile card, expanding sub-nav.
  → PropertyRow (Discord variant): label ABOVE value, not beside it.
    Different anatomy from Notion's SettingRow — both are extractable.
  → ToggleRow (Discord variant): label + optional description + toggle.
    Simpler than Notion's (no section grouping within the row).
  → SwatchSelector: grid of color/theme tiles with selection state.
    Unique to Discord — not seen in Notion.
  → LivePreview: embedded rendered preview of settings changes.
  → PromotionalCard: illustration + text + CTA within settings.
  → CrossLink row: icon + title + description + chevron, linking to
    related settings.
  → Sub-navigation within settings sections (expand/collapse in sidebar).

2. COMPOSITIONAL PATTERNS

------------------------------------------------------------------------
2A. MESSAGE ROW
------------------------------------------------------------------------
Sources: #rdc-gang, #variety-fighters channels

The most frequently repeated pattern in Discord. Equivalent to Notion's
SettingRow in terms of repetition frequency and extraction value.

Full message anatomy (first message in a user group):
  +-------------------------------------------------------------+
  | [Avatar]  [Username] [Role badges] [Timestamp]               |
  |           [Message body text]                                 |
  |           [Optional: embedded image/GIF/video]                |
  |           [Optional: reaction badges row]                     |
  +-------------------------------------------------------------+

  - Avatar: circular, ~40px, left-aligned. Shows user's profile
    picture or default colored avatar with initial.
  - Username: colored text — the color corresponds to the user's
    highest role in the server. Clickable (opens user profile popover).
  - Role badges: small pill badges appearing after the username.
    Observed types:
    * Colored dot + text badges (e.g., "RAGE" in green, "ZLDA" in
      blue, "CLUB" in orange, "RP18" in red, "FATE" in teal).
    * Emoji badges (small emoji icons inline with username).
    * "APP" badge (blue, for bot/application messages like Mee6).
    * "BOT" badge (blue, similar to APP).
  - Timestamp: muted text, right of username/badges. Format:
    "12:15 AM" for today, "Yesterday at 1:20 PM" for yesterday.
  - Message body: white/primary text below the username line. Can
    contain: plain text, emoji (both custom server emoji and standard
    Unicode), @mentions (highlighted in blue), inline links.
  - "(edited)" indicator: muted text appended to edited messages.

Condensed message (subsequent message from same user):
  +-------------------------------------------------------------+
  |           [Message body text]         [Timestamp on hover]    |
  +-------------------------------------------------------------+

  - No avatar, no username — just the body text.
  - Timestamp appears on hover (left of the message, in the space
    where the avatar would be).

Reply chain (message replying to another):
  +-------------------------------------------------------------+
  | [Arrow icon] [Mini avatar] @[Username] [Role badge] [Quote]  |
  | [Avatar]  [Username] [Role badges] [Timestamp]               |
  |           [Reply body text]                                   |
  +-------------------------------------------------------------+

  - Reply reference line sits above the full message anatomy.
  - Small reply arrow icon (curved) + miniature avatar (~16px) +
    @username in blue + role badge + truncated text of the original
    message.
  - Clicking the reply reference scrolls to the original message.

Bot/system message:
  - Same anatomy as user messages but username has "APP" or "BOT"
    badge.
  - Bot messages can contain: embed cards (rich formatted blocks
    with colored left border), buttons, and interactive components.
  - Observed: Mee6 bot with level-up notification message.

Extraction potential for Neighborhood:
  → MessageRow: the full message component with all its variants
    (full, condensed, reply, bot). This is a complex composed component.
  → ReplyReference: the quoted reference line above a reply.
  → The message grouping logic (first message shows full anatomy,
    subsequent messages from same user condense) is an interaction
    pattern, not just a visual one.

2B. USER PROFILE POPOVER
------------------------------------------------------------------------
Source: Left-clicking a username in chat

The richest single popover component observed across all three products.

Anatomy (top to bottom):
  +-------------------------------------------------------------+
  | [Banner image — full width, ~100px tall]                     |
  |                                                              |
  | [Large avatar (~80px) overlapping banner bottom edge]        |
  | [Status indicator dot on avatar]                             |
  |                                                              |
  | [Display name (large, bold)] [Action icons: ...]             |
  | [Username + connection info (muted)]                         |
  | [Activity/badge icon row]                                    |
  |                                                              |
  | [Mutual servers: icon + "1 Mutual Server"]                   |
  |                                                              |
  | [Bio text (regular weight, wrapping)]                        |
  |                                                              |
  | [Role badges: colored dot + role name] [+17 overflow]        |
  |                                                              |
  | [Message input: "Message @[Name]" + emoji icon]              |
  +-------------------------------------------------------------+

  - Banner image: custom user banner, full width of popover.
  - Avatar: large (~80px), circular, overlapping the banner's bottom
    edge by ~50%. Has a status indicator dot (green = online,
    yellow = idle, red = DND, gray = offline) at the bottom-right.
  - Display name: large, bold. May have custom font/styling for
    Nitro users.
  - Three-dot action menu icon beside the name.
  - Username: muted text, with platform connection metadata
    (e.g., "daisaku_kuze • Kat" showing connected accounts).
  - Activity badges: row of small emoji-sized icons representing
    current activities, connections, or decorations.
  - Mutual servers: server icon + "1 Mutual Server" text.
  - Bio: user-written text, regular weight. Can contain custom
    content (the observed bio contained Latin-looking placeholder text).
  - Role badges: horizontal wrapping row of pill badges. Each badge:
    colored dot (matching role color) + role name text. When many
    roles exist, the row shows visible badges + "+17" overflow count.
  - Message input: inline text input at the bottom of the popover.
    Placeholder: "Message @[username]". Emoji button on the right.
    This allows messaging the user directly from the popover without
    navigating to DMs.

Extraction potential for Neighborhood:
  → UserProfileCard: a rich composed popover. Contains avatar with
    status, identity block, badge rows with overflow, bio section,
    and inline input. High demonstration value.
  → AvatarWithStatus: avatar component with status indicator dot.
    Discord's avatar is more complex than Notion's (which has no
    status indicator).
  → RoleBadgeRow: horizontal wrapping row of pill badges with
    overflow count. The overflow "+N" pattern is reusable beyond
    just roles.
  → InlineInput: text input embedded within a card/popover, used
    for quick actions without navigation.

2C. CONTEXT MENU (THREE VARIANTS)
------------------------------------------------------------------------
Sources: Right-click on username, right-click on message, server
  name dropdown

Discord has THREE distinct context menu patterns. All share the same
base structure but contain different content types.

Shared anatomy:
  - Dark surface, subtle rounded corners.
  - Items: icon + label text, full-width row, highlight on hover.
  - Dividers: subtle horizontal lines between groups.
  - Submenu indicator: chevron ">" on right side.
  - Destructive items: red text (e.g., Block, Leave Server, Report).

Variant 1 — User context menu (right-click username):
  +-------------------------------------------+
  | Profile                                    |
  | Message                                    |
  | Start a Call                               |
  | Add Note                                   |
  |   Only visible to you  <- description      |
  |-------------------------------------------|
  | Apps                                    >  |
  | Invite to Server                        >  |
  | Add Friend                                 |
  | Ignore                                     |
  | Block                    <- RED            |
  |-------------------------------------------|
  | Roles                                   >  |
  |-------------------------------------------|
  | Copy User ID                          [📋] |
  +-------------------------------------------+

  Notable: "Add Note" has a description line below it ("Only visible
  to you") — the only item with a description in this menu. "Block"
  is styled in red. "Copy User ID" has a clipboard icon.

Variant 2 — Message context menu (right-click message):
  +-------------------------------------------+
  | [😂] [👍] [🔥] [🤚] [⚙]  <- emoji row    |
  |-------------------------------------------|
  | Add Reaction                            >  |
  |-------------------------------------------|
  | Reply                                  [↩] |
  | Forward                                [↗] |
  |-------------------------------------------|
  | Copy Text                              [📋]|
  | Apps                                    >  |
  |-------------------------------------------|
  | Mark Unread                            [📧]|
  | Copy Message Link                      [🔗]|
  | Speak Message                          [🔊]|
  |-------------------------------------------|
  | Report Message                    <- RED [🚩]|
  |-------------------------------------------|
  | Copy Message ID                        [🆔]|
  +-------------------------------------------+

  Notable: Emoji quick-reaction row at the top — 4 recently-used
  emoji as circular buttons + a gear/settings icon to customize.
  This is unique to message context menus. Every action item has
  a right-aligned icon. "Report Message" is red/destructive.

Variant 3 — Server dropdown (click server name):
  +-------------------------------------------+
  | Server Boost                          [🚀] |
  | Server Tag                     [🎮 -RDC]  |
  |-------------------------------------------|
  | Invite to Server                      [👤+]|
  | App Directory                         [📱] |
  |-------------------------------------------|
  | Show All Channels                     [☐]  |
  | Notification Settings                 [🔔] |
  | Privacy Settings                      [⚙]  |
  |-------------------------------------------|
  | Edit Per-server Profile               [✏]  |
  | Hide Muted Channels                   [☐]  |
  |-------------------------------------------|
  | Leave Server                   <- RED [🚪] |
  +-------------------------------------------+

  Notable: "Show All Channels" and "Hide Muted Channels" have
  checkbox toggles (not just action items). "Server Tag" shows
  a custom emoji badge inline. "Leave Server" is red/destructive.

Extraction potential for Neighborhood:
  → ContextMenu base component with slots for different content.
  → MenuItem variants: action (icon + label), destructive (red),
    with-description (label + muted text below), with-checkbox,
    with-submenu (chevron), with-badge (custom content right-aligned).
  → EmojiQuickBar: horizontal row of emoji action buttons, unique
    to Discord's message context menu.
  → Three variants from one product is strong evidence that menus
    need a flexible, composable architecture — not a single rigid
    component.

2D. SETTING ROW (DISCORD VARIANT)
------------------------------------------------------------------------
Sources: Settings > Notifications, Settings > Appearance

Discord's setting row pattern is DIFFERENT from Notion's.

Notion's SettingRow:
  [Label + Description (left)]  [Control (right)]
  — label and description are side-by-side with the control

Discord's SettingRow variants:

Toggle variant:
  +-------------------------------------------------------------+
  | [Label text (bold/primary)]                     [Toggle]     |
  | [Description text (muted, below label)]                      |
  +-------------------------------------------------------------+

  - Label is primary text, bold or medium weight.
  - Description below the label, muted. Can wrap to multiple lines.
  - Toggle right-aligned, vertically centered against the label
    (not the full label+description block).
  - Observed: "Enable Desktop Notifications", "People I know start
    streaming in small servers", etc.
  - Some toggle rows have an additional action link below the
    description: "Preview Sound" (blue link text).

Select variant:
  +-------------------------------------------------------------+
  | [Label text]                          [Select dropdown]      |
  +-------------------------------------------------------------+

  - Simpler — label only, no description. Select right-aligned.
  - Observed: "Someone reacts to my messages" + "All Messages ▾"

Property row variant (My Account page):
  +-------------------------------------------------------------+
  | [Label text (muted, small)]                                  |
  | [Value text (primary)]                        [Edit button]  |
  +-------------------------------------------------------------+

  - Label ABOVE value, not beside it.
  - Value is the primary visual element.
  - Action buttons right-aligned: "Edit", or "Remove" + "Edit",
    or "Reveal" link inline with the value.
  - Masked values: "********@gmail.com" with "Reveal" link in blue.
  - This is structurally different from both Notion's SettingRow
    and Discord's toggle variant.

Cross-link variant:
  +-------------------------------------------------------------+
  | [Icon]  [Title (bold)]                              [>]      |
  |         [Description (muted)]                                |
  +-------------------------------------------------------------+

  - Observed: "Related Settings" section at bottom of Notifications.
  - Icon prefix, title + description, chevron on right.
  - Clicking navigates to another settings page.
  - Similar to a navigation item but within the content area.

Extraction potential for Neighborhood:
  → SettingRow needs multiple variants to cover both Notion and
    Discord patterns. The core abstraction is: label + optional
    description + right-aligned control slot. But the layout
    relationship between label, description, and control differs.
  → PropertyRow (Discord): label above value with action buttons.
    Distinct from Notion's icon + label + value pattern.
  → CrossLinkRow: navigational item within settings content.

2E. REACTION BADGES
------------------------------------------------------------------------
Source: Messages in #rdc-gang

Anatomy:
  [😂 1]  [👏 1]  [😊]

  - Each reaction: emoji + count, inside a pill-shaped badge.
  - Badge has subtle border, slightly tinted background.
  - Active reaction (user has reacted): border color accent or
    tinted background indicating participation.
  - "Add reaction" button: smiley face icon with "+" or just a
    smiley icon, appears at the end of the reaction row.
  - Multiple reactions display horizontally, wrapping if needed.
  - Clicking a reaction toggles your own reaction on/off.
  - Count updates live.

Extraction potential for Neighborhood:
  → ReactionBadge: emoji + count, pill shape, active/inactive states.
  → ReactionRow: horizontal wrapping row of ReactionBadges + add
    action. Could be abstracted as a TagGroup or BadgeGroup with
    an add action.

------------------------------------------------------------------------
2F. MESSAGE INPUT BAR
------------------------------------------------------------------------
Source: Bottom of any text channel

Anatomy:
  +-------------------------------------------------------------+
  | [+]  [Message #channel-name 🤟]           [🎁][GIF][🎭][😊] |
  +-------------------------------------------------------------+

  - Plus (+) icon: left side, opens file/media attachment menu.
  - Text input: large, with placeholder "Message #[channel-name]"
    that includes the channel emoji. Multiline capable.
  - Right-side action icons: gift (Nitro gift), GIF (GIF picker),
    sticker (sticker picker), emoji (emoji picker).
  - Border: subtle rounded border on the entire bar.
  - Typing in the input shows the text and hides the placeholder.
  - Above the input (when active): formatting toolbar may appear
    for rich text editing.

Extraction potential for Neighborhood:
  → MessageInput / ComposerBar: input with action icons on both
    sides. More complex than Notion's comment input (which is just
    avatar + plain input). The action icon row is a pattern itself.

------------------------------------------------------------------------
2G. CHANNEL HEADER BAR
------------------------------------------------------------------------
Source: Top of any channel view

Anatomy:
  +-------------------------------------------------------------+
  | # [channel-name] 🤟 | [description text]    [icons: 📌🔔👥🔍]|
  +-------------------------------------------------------------+

  - Left: hash (#) icon + channel name + optional emoji + vertical
    divider + channel description/topic in muted text.
  - Right: row of icon-only action buttons — threads, notification
    bell, pinned messages, member list toggle, search magnifier,
    inbox, help.
  - The description text is truncated if too long.
  - For DM conversations: shows user avatar + name instead of # icon.

Extraction potential for Neighborhood:
  → ChannelHeader / ViewHeader: icon + title + metadata + action
    icons. Compare to Notion's top bar — same concept (contextual
    header for current view) but different anatomy.

------------------------------------------------------------------------
2H. USER PANEL (BOTTOM BAR)
------------------------------------------------------------------------
Source: Bottom-left of app, below channel sidebar

Anatomy:
  +-------------------------------------------------------------+
  | [Avatar + status dot]  [Display name]   [🎤] [🎧] [⚙]      |
  |                        [Status text]                         |
  +-------------------------------------------------------------+

  - Avatar: small (~32px), circular, with status indicator dot.
  - Display name: primary text.
  - Status: "Online", "Invisible", custom status — muted text below.
  - Three icon buttons: microphone (mute/unmute), headphones
    (deafen/undeafen), gear (settings). These buttons toggle state
    — when muted, the mic icon shows a red slash through it.
  - Always visible — never hidden or collapsible.
  - Clicking the avatar/name area opens a status/profile popover.

Extraction potential for Neighborhood:
  → UserPanel / IdentityBar: compact user identity display with
    quick-access toggle controls. Unique to Discord — Notion has
    no equivalent persistent user panel.

------------------------------------------------------------------------
2I. PROMOTIONAL / UPSELL CARD
------------------------------------------------------------------------
Source: Settings > Appearance (Nitro upsell)

Anatomy:
  +-------------------------------------------------------------+
  | [Illustration]  [Title: "Make your Discord look *juuusst*    |
  |                  right"]                                     |
  |                 [Description text]                           |
  |                 [CTA Button: "Try it out" (blue, outlined)]  |
  +-------------------------------------------------------------+

  - Horizontal layout: illustration on left, text + CTA on right.
  - Appears inline within settings content, not as a modal or banner.
  - Rounded corners, subtle background differentiation.
  - Used for premium feature upselling.

Extraction potential for Neighborhood:
  → PromotionalCard / FeatureCard: illustration + text + CTA. Could
    be useful for a design system docs site to highlight features
    or link to related resources.

2J. SWATCH SELECTOR GRID
------------------------------------------------------------------------
Source: Settings > Appearance (theme and color selection)

Theme swatch row:
  [■ Dark] [■ Light] [■ Darker] [■ Midnight] [🔄 Sync]

  - 4 square swatches + 1 sync toggle icon.
  - Selected swatch: visible highlight/border.
  - Each swatch is a clickable square with a dark fill representing

Color theme grid:
  - ~20 gradient color swatches arranged in 3 rows.

Extraction potential for Neighborhood:

3. ATOMIC COMPONENTS
========================================================================

------------------------------------------------------------------------
3A. BUTTON
------------------------------------------------------------------------
Variants observed:
  1. Primary (filled): Blue/purple background, white text. "Change
     Password", "Enable Authenticator App", "Preview Themes",
     "+ Add to Channel List". Used for primary actions.
  2. Secondary (outlined): Subtle border, no fill. "Edit User
     Profile", "Edit" buttons in account settings. Used for
     secondary actions.
  3. Ghost/text: No background, no border. "Reveal" links in
     masked settings values, "Preview Sound" links. Behaves like
     a link but within a button context.
  4. Destructive: Red text. "Leave Server", "Block", "Report
     Message". Used for dangerous/irreversible actions.
  5. Icon-only: Mic, deafen, settings icons in user panel.
     Toolbar icons in channel header. Muted by default.
  6. Split/compound: "Mark As Read" text + dismiss icon in the
     unread messages banner.

States observed:
  - Default, Hover (lighter/brighter), Active, Disabled (Nitro-
    locked features show disabled state)

3B. TOGGLE / SWITCH
------------------------------------------------------------------------
Observed in: Settings > Notifications (multiple rows)

  - Track: rounded pill shape, similar to Notion's.
  - Off state: gray track, circle thumb on left.
  - On state: blue/purple track, circle thumb on right.
  - Visually identical to Notion's toggle — same pattern.
  - No size variants observed — single size throughout.

------------------------------------------------------------------------
3C. SELECT / DROPDOWN
------------------------------------------------------------------------
Observed in: Settings > Notifications ("All Messages" dropdown)

  - Trigger: text value + downward chevron.
  - Background: slightly lighter than the page surface.
  - Border: subtle.
  - Dropdown content not directly captured (would need to click open).

3D. CHECKBOX
------------------------------------------------------------------------
Observed in: Server dropdown (Show All Channels, Hide Muted Channels)

  - Square shape, embedded within dropdown menu items.
  - Checked/unchecked states — no intermediate state observed.
  - Used for boolean toggles within menus (not in forms).

------------------------------------------------------------------------
3E. BADGE / PILL
------------------------------------------------------------------------
Multiple distinct variants observed:

  1. Role badge: colored dot + role name text, pill shape.
     Examples: green dot + "RAGE", blue dot + "ZLDA", orange dot +
     "CLUB". Appears after usernames and in profile popovers.
  2. Notification count badge: red circle with white number.
     Appears on server icons and channel items. Sizes vary based on
     digit count.
  3. "NEW" badge: pink/red pill with "NEW" text. Appears on settings
     nav items (Chat) and appearance swatches.
  4. "APP" / "BOT" badge: blue pill with text. Appears next to bot
     usernames in messages.
  5. Overflow count badge: "+17" — gray pill showing hidden count.
     Appears at end of role badge row.
  6. Server tag badge: custom emoji + text (e.g., "🎮 -RDC") in
     server dropdown. Special treatment with custom emoji rendering.

  Discord has the most badge variety of the three products.

3F. AVATAR
------------------------------------------------------------------------
Most complex avatar system of the three products.

Variants:
  1. Server icon: circular (~48px), image or text initials, in the
     server strip. Transitions to rounded square on hover.
  2. User avatar (message): circular (~40px), left of message body.
  3. User avatar (popover): large (~80px), overlapping banner image.
  4. User avatar (mini): circular (~16px), in reply reference lines
     and typing indicators.
  5. User avatar (user panel): circular (~32px), with status dot.
  6. User avatar (settings): large, in profile card at top of
     settings sidebar.

Status indicator:
  - Small dot at bottom-right of avatar circle.
  - Green: online. Yellow: idle. Red: DND. Gray: offline/invisible.
  - Only appears on user avatars, not server icons.

Active server indicator:
  - White vertical pill on the left edge of the server icon.
  - Full height (~40px) for active, shorter (~20px) for hovered,
    even shorter (~8px) for unread notification.

3G. TABS

Observed in: Settings > My Account (Security / Standing)

  - Inactive tab: muted text, no underline.
  - Positioned at top of the content area.
  - Only one tab style observed in Discord (vs. Notion's three).

3H. DIVIDER / SEPARATOR

Observed in: Context menus, settings rows, channel sidebar categories

  - Horizontal line, very subtle (1px, low opacity).

4. INTERACTION PATTERNS
========================================================================

4A. MESSAGE HOVER TOOLBAR
  Hovering over a message reveals a floating toolbar at the top-right
  of the message row. Contains: quick reaction emoji buttons (4
  recently-used) + action icons (reply, thread, more). The message
  row also gets a subtly lighter background on hover. Similar concept
  to Notion's sidebar hover reveal but implemented as a floating
  toolbar rather than inline icons.

4B. MESSAGE GROUPING
  Messages from the same user within a short time window are grouped.
  The first message shows full anatomy (avatar + username + timestamp
  + body). Subsequent messages show only the body text, with the
  timestamp revealed on hover in the avatar space. This reduces visual
  repetition in active conversations.

4C. SERVER ICON INDICATORS
  Three levels of visual state on server icons:
  - Active: full white left border pill.
  - Hovered: medium white left border pill.
  - Unread: small white dot / short border pill.
  Plus red notification badge with count overlapping the icon.

4D. CATEGORY COLLAPSE
  Channel categories (e.g., "ENTERTAINMENT", "Social VC's") are
  collapsible. Clicking the category header toggles the visibility
  of all channels within it. The chevron rotates to indicate state.
  This is the same collapsible section pattern seen in Notion's
  sidebar but applied to channel groups.

4E. FULL-PAGE SETTINGS TAKEOVER
  Unlike Notion (which uses a modal overlay), Discord's settings
  completely replaces the app shell. The ESC key or X button returns
  to the app. This is a stronger separation — settings is a "mode"
  you enter, not a layer on top.

4F. LIVE PREVIEW
  In Appearance settings, a miniature chat view renders at the top
  showing how the current theme looks. This updates in real-time as
  the user changes theme selections. A unique interaction pattern
  not seen in Notion.

4G. INLINE MESSAGING FROM POPOVER
  The user profile popover includes a message input at the bottom.
  Users can type and send a DM directly from the popover without
  navigating to a DM channel. This is "action in context" — reducing
  navigation by embedding actions where the intent naturally arises.

4H. TYPING INDICATOR
  Animated three-dot indicator appears when another user is typing.
  Shows the username(s). This is a real-time presence pattern unique
  to communication apps.

4I. UNREAD SCROLL BANNER
  When entering a channel with unread messages, a yellow banner
  appears at the top: "13+ new messages since [date]". Clicking it
  scrolls to the first unread message. "Mark As Read" action on the
  right dismisses it. This is a navigational aid within a scrollable
  feed.


========================================================================
5. NOTABLE DESIGN DECISIONS
========================================================================

5A. COLOR AS IDENTITY
  Discord uses color extensively for identity — each role has a color,
  and usernames in chat display in their role color. This creates a
  visual hierarchy where active participants become recognizable by
  color before you read their name. Notion uses color minimally
  (primarily for status badges).

5B. HEAVY BADGE USAGE
  Discord has at least 6 distinct badge types (role, notification
  count, NEW, APP/BOT, overflow count, server tag). Badges carry
  significant informational load — they communicate role, status,
  unread state, feature newness, and bot identity all through the
  same general pattern with different visual treatments.

5C. CONTEXT MENUS AS PRIMARY INTERACTION
  Discord relies heavily on right-click context menus as the primary
  way to access actions. This is different from Notion (which uses
  three-dot dropdowns clicked from visible icons) and more aligned
  with desktop application conventions. Three distinct context menu
  variants exist for different object types.

5D. PERSISTENT USER PANEL
  The user panel at the bottom-left is always visible — even during
  settings. It provides constant access to mic/deafen controls and
  user status. This is a trust-building pattern for a communication
  app: you always know your audio state.

5E. REAL-TIME INDICATORS EVERYWHERE
  Typing indicators, online status dots, voice channel user counts,
  unread badges, message timestamps — Discord surfaces real-time
  state constantly. This density of live information is unique among
  the three audited products.

5F. THREE-COLUMN LAYOUT IS NON-NEGOTIABLE
  Unlike Notion (where the sidebar can collapse), Discord's three-
  column layout is persistent. The server strip and channel sidebar
  are always visible. This prioritizes navigation speed over content
  space — appropriate for a communication app where switching
  contexts is the primary interaction.

5G. UPSELL INTEGRATED INTO SETTINGS
  Nitro promotional cards appear inline within settings content
  (not as separate modals or banners). Color theme swatches show
  locked states for Nitro-only options. This "freemium signal" pattern
  is unique to Discord among the three audited products.

5H. SUB-NAVIGATION WITHIN SETTINGS SECTIONS
  Settings sections like Notifications and Appearance have their own
  sub-navigation (Overview, Sounds, Badges, etc.) that appears as
  indented items in the sidebar. This creates a three-level hierarchy:
  section group > section > sub-section. Notion only has two levels.
